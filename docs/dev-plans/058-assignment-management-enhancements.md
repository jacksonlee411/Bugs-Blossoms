# DEV-PLAN-058：任职管理增强（对齐 050 §10，051 阶段 F）

**状态**: 规划中（2025-12-20 01:46 UTC）

## 1. 背景
- 050 §10 列出“任职管理后续能力清单”（多段任职事件、未来计划任职、历史更正审计、任职类型：主任职/兼任/代理等）。
- 051 将其明确为后续排期的独立里程碑；本计划用于拆解可交付的增量路线，避免阻塞 Position Core 上线。

## 2. 工具链与门禁（SSOT 引用）
- 与 Org 时间线/审计/冻结窗口体系保持一致：见 `docs/dev-plans/025-org-time-and-audit.md`。
- 门禁触发与本地必跑以 `AGENTS.md` 为准。

## 3. 目标（完成标准）
- [ ] 在不破坏 v1（053）的前提下，逐步引入“任职事件化/多段任职/计划任职/任职类型”等能力。
- [ ] 明确每个增量的 schema 影响面、迁移策略、权限边界与可回滚方案（与 DEV-PLAN-059 收口对齐）。

## 4. 依赖与并行
- 依赖：`DEV-PLAN-053`（Assignment v1）。
- 可并行：在 v1 上线后逐步迭代；不作为 v1 上线前置条件。

## 5. 实施步骤（建议拆分为独立增量）
1. [ ] 增量 1：任职类型（主任职/兼任/代理）
   - 定义类型枚举、默认规则、可见性与授权边界；不改变核心占编口径（或明确口径差异）。
2. [ ] 增量 2：未来计划任职（planned assignments）
   - 明确与 Position 生命周期/填充状态的联动口径（例如未来计划是否影响 vacancy）。
3. [ ] 增量 3：多段任职事件（segments）
   - 明确“同一 subject 在同一 Position 的时间线”如何表示（事件表/切片表/组合策略），以及更正/撤销的审计口径。
4. [ ] 增量 4：历史更正审计增强
   - 对齐 Correct 权限边界与审计追溯：谁在何时改了哪个历史版本、原因是什么、影响范围如何复盘。
